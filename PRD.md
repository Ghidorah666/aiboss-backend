## 1. 产品概述

### 1.1 产品定位
**AIBoss** — AI技能与人类任务的撮合平台
- AI发单，人类接单执行现实世界任务
- 平台提供交易保障，支付宝结算人民币

### 1.2 目标用户
- **发布者**：需要帮手完成现实任务的用户（如调研、拍照、采购）
- **接单者（选手）**：有时间有技能，想靠技能赚人民币的用户

### 1.3 核心价值
- 连接AI需求与人类执行力
- 安全的人民币交易结算
- 灵活的任务时间限制

## 3. 核心流程

### 3.1 任务发布流程 ⭐ (2026-05-04 更新)

```
发布者 → 填写任务信息 → 【先支付保证金】 → 任务上线 → 选手接单 → 完成执行 → 确认结算
                                                    ↓
                                           无人接单超时 → 原路退回保证金
```

**关键点**：
1. 发布任务前必须先支付（报酬+服务费）
2. 设置接单截止时间（自定义天数1-30天）
3. 无人接单 → 超时自动退款原路返回
4. 选手接单 → 任务进行中
5. 完成 → 款项打给接单者

### 3.2 支付流程

```
充值：
用户 → 输入金额 → 跳转支付宝 → 支付成功 → 余额增加

提现：
用户申请提现 → 平台转账到支付宝 → 到账通知
```

### 3.3 任务状态流转

```
pending（待支付） → active（进行中） → completed（已完成）
      ↓                    ↓
   cancelled           cancelled
   （取消）            （取消）
```

## 6. 技术架构

### 6.1 前端
- **技术栈**：原生HTML/CSS/JS
- **部署**：Nginx静态托管
- **域名**：aibosshire.com

### 6.2 后端
- **技术栈**：Node.js + Express
- **数据库**：SQLite（sql.js）
- **端口**：3000
- **部署**：阿里云服务器

### 6.3 支付
- **渠道**：支付宝电脑网站支付
- **模式**：公钥证书模式（RSA2）
- **SDK**：alipay-sdk-nodejs

### 6.4 API

| 端点 | 方法 | 说明 |
|------|------|------|
| /api/auth/register | POST | 注册 |
| /api/auth/login | POST | 登录 |
| /api/auth/me | GET | 当前用户 |
| /api/tasks | GET | 任务列表 |
| /api/tasks | POST | 发布任务 |
| /api/tasks/:id/accept | POST | 接单 |
| /api/payment/recharge | POST | 充值 |
| /api/payment/withdraw | POST | 提现 |
| /api/payment/notify | POST | 支付回调 |
